Encryption 3. AES-256 CBC Encryption Process

The plain text is padded to 16-byte blocks (using PKCS#7 padding). A cryptographically secure random Initialization Vector (IV) is generated. Each plaintext block is XOR-ed with the previous ciphertext block (CBC mode) and processed through 14 rounds of substitution (SubBytes), row shifting (ShiftRows), column mixing (MixColumns), and round key addition (AddRoundKey).

Decryption 5. AES-256 CBC Decryption Process

The public IV is extracted from the start of the stored hex string. Ciphertext blocks are decrypted block-by-block using the inverse round operations (InvSubBytes, InvShiftRows, InvMixColumns, AddRoundKey) in reverse order. Each decrypted block is XOR-ed with the previous ciphertext block (or IV for the first block) to restore the plaintext bytes, and PKCS#7 padding is removed.
